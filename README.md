# 能源动力保研信息追踪网页

这是一个适合 GitHub Pages 的静态网页。现在采用最稳定的维护方式：

```text
每天编辑 data.csv -> 上传/提交到 GitHub -> 网页自动读取 data.csv
```

不再依赖 Google Sheets、金山文档、腾讯文档或飞书表格，所以微信浏览器和国内手机网络访问会稳定很多。

## 文件说明

- `index.html`：网页代码，通常不用每天改。
- `data.csv`：通知数据，每天只需要更新这个文件。

两个文件都必须放在 GitHub 仓库根目录，也就是打开仓库后第一层能直接看到的位置。

## CSV 表头

`data.csv` 第一行不要修改，必须是：

```text
school,program,category,deadline,link,city,priority,note
```

字段含义：

- `school`：学校或研究院名称，例如 `清华大学`
- `program`：学院、项目、研究所或通知名称
- `category`：信息类型，例如 `夏令营`、`预推免`、`九推`、`直博`
- `deadline`：截止时间，推荐格式 `2026-07-01 23:59`
- `link`：通知链接
- `city`：城市，可留空
- `priority`：优先级，可写 `高`、`中`、`低`
- `note`：备注，可留空

如果不知道截止时间，`deadline` 写：

```text
N/A
```

## 每天更新流程

1. 打开 GitHub 仓库里的 `data.csv`。
2. 点右上角铅笔按钮编辑。
3. 新增、删除或修改通知行。
4. 点击 `Commit changes`。
5. 等 1 到 5 分钟后刷新网页。

网页读取 `data.csv` 时加了时间参数，能减少旧缓存导致的“不更新”问题。

## 注意事项

- 不要删除第一行表头。
- 每一行逗号数量要一致。
- 如果某个单元格里必须写英文逗号，请用英文双引号包起来，例如 `"材料,能源交叉方向"`。
- 公开发布前请核对通知来源、截止时间和招生要求。
